# 自动评教助手 使用指南

面向 NJUST 学生评教场景的自动化辅助模块（ACE, Auto Course Evaluation）。它提供安全可控的“一键评教/批量提交”能力，同时保留手动精调与复核环节，兼顾效率与准确性。

---

## 适用范围

- 页面：学生评教入口（包含类别列表页与课程列表页）
- 对象：当前学期或历史学期可评的课程（以页面展示为准）
- 浏览器：现代浏览器（Chrome/Edge 等内核），需允许脚本正常运行与弹窗打开

---

## 功能亮点

- 批量处理：对勾选的课程依次执行“进入 → 填分 → 保存/提交”
- 两种模式：手动模式（可微调每项得分）与自动模式（默认全项最高分）
- 安全策略：涉及“提交”时始终要求用户手动确认，不进行自动点击
- 可视化日志：底部“增强助手运行日志”实时输出执行过程与结果
- 可回溯：先“保存为待提交”，核对后再集中提交，降低误操作风险

## 快速上手

1. 进入评教入口页（“类别”列表页）。
2. 点击任一类别卡片，进入该类别的“课程列表页”。
3. 在课程列表页：
   - 勾选需要自动处理的课程（默认全选）。
   - 点击“开始评价并保存”以批量填分并保存为“待提交”。
   - 核对无误后，点击“提交已评课程”批量提交。
4. 如需单独核对，可使用每行的“查看”按钮在新窗口打开评价页。
5. 如果需要清理状态：
   - “重置缓存”会清空本模块的本地缓存与进度标记；
   - “清空日志”仅清理底部运行日志，不影响已保存/提交记录。

---

## 模式与策略

- 手动模式
  - 提供四档快捷分数（最高；高；中；低）
  - 页面实时显示单项得分与总分，可逐题微调后再保存/提交。
- 自动模式
  - 默认策略为“全项最高分”；遇互斥题按“去重优先”选择合理项。
  - 对勾选课程按顺序执行“填分 → 保存为待提交”。

建议流程：先批量“保存为待提交”，再集中“提交已评课程”。

---

## 安全与确认

- 涉及“提交”的操作会弹出浏览器原生确认框；必须由用户亲自点击“确认”按钮后才会提交。



## 日志与状态面板

底部“增强助手运行日志”用于实时展示执行细节：

- 分级标签：
  - [ERR] 错误（失败或不可恢复问题）
  - [WRN] 警告（可继续但需关注）
  - [INF] 信息（关键进度提示）
  - [OK ] 成功（关键步骤完成）
  - [DBG] 调试（详细过程）
- 标题栏状态：显示最近一条事件摘要，5 秒无新日志自动复位为“南理工教务增强助手V2”。
- “清空”可快速清理列表，不影响实际处理进度。
